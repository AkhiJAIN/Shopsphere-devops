# Objective

The objective of this project is to deploy a production-ready, cloud-native e-commerce application on AWS by implementing modern DevOps practices. The infrastructure will be provisioned using Infrastructure as Code (Terraform), configured with Ansible, containerized using Docker, orchestrated with Kubernetes (Amazon EKS), automated through Jenkins CI/CD pipelines, and monitored using Prometheus, Grafana, and AWS CloudWatch.

---

# Project Overview

ShopSphere is an enterprise-grade e-commerce application consisting of three logical layers:

* **Presentation Layer:** React.js application served through Nginx.
* **Application Layer:** Spring Boot REST API responsible for business logic.
* **Data Layer:** MySQL database hosted on Amazon RDS for persistent data storage.

The application is designed for scalability, high availability, security, and automated deployments.

---

# Architecture Overview

The project follows the Three-Tier Architecture pattern.

```
Users
   │
   ▼
React + Nginx (Frontend)
   │
REST API
   │
Spring Boot Backend
   │
Amazon RDS MySQL
```

# Network Design

A highly available network architecture will be deployed across two Availability Zones.

The planned network includes:

* One Virtual Private Cloud (VPC)
* Two Public Subnets
* Two Private Application Subnets
* Two Private Database Subnets
* Internet Gateway
* NAT Gateway
* Route Tables
* Security Groups
